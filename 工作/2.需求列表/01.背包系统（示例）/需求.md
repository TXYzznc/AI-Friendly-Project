## 需求详情

### R1: 背包数据模型

**描述**：实现背包的核心数据结构和容量管理

**技术需求**：
- 创建 `InventoryModule` (IGameModule)
- 物品栏数据结构：`List<InventorySlot>`（每个 Slot 包含物品ID、数量）
- 背包容量：100 格（可配置）
- 物品叠放：相同物品可堆放（最多 999 个）
- 事件：物品增加、物品减少、背包满（EventBus）

**API**：
```csharp
// 添加物品
bool AddItem(int itemId, int count);

// 移除物品
bool RemoveItem(int itemId, int count);

// 获取背包信息
InventoryData GetInventoryData();
```

**数据结构**：
```csharp
public class InventorySlot
{
    public int itemId;
    public int count;
}

public class InventoryData
{
    public List<InventorySlot> slots;
    public int maxCapacity;
}
```

---

### R2: 背包 UI

**描述**：背包窗口的 UI 展示和交互

**技术需求**：
- 创建 `InventoryUIForm`（UI Form）
- 背包窗口（4x5 = 20 格展示，支持滚动显示全部 100 格）
- 物品图标显示 + 数量标签
- 选中物品时显示详细信息面板
- 打开/关闭背包窗口

**功能**：
- 物品网格显示（GridLayoutGroup）
- 物品选中高亮
- 详细信息面板（名称、描述、属性）
- 关闭按钮

**事件**：
- 物品选中事件
- 背包窗口打开/关闭事件

---

### R3: 物品系统

**描述**：物品信息定义和管理

**技术需求**：
- 物品 DataTable（Excel）：ItemTable.xlsx
- 字段：ItemId, ItemName, ItemType, Rarity, Icon, Description, MaxStack
- ItemManager 模块：提供物品信息查询接口
- 物品类型枚举：Weapon, Armor, Consumable, Material, ...

**接口**：
```csharp
ItemInfo GetItemInfo(int itemId);  // 获取物品信息
Sprite GetItemIcon(int itemId);     // 获取物品图标
```

---

### R4: 背包逻辑

**描述**：物品添加、删除、使用的具体逻辑

**技术需求**：

1. **物品添加**
   - 检查背包容量
   - 如果已有该物品，增加数量
   - 如果没有，新增一个 Slot
   - 满了则返回失败，剩余物品堆放到地面

2. **物品删除/丢弃**
   - 检查物品是否存在
   - 减少数量或删除 Slot
   - 发布物品减少事件

3. **物品使用**
   - 仅限消耗品
   - 执行物品的效果（可扩展）
   - 减少数量

**事件**：
- ItemAdded
- ItemRemoved
- ItemUsed

---

### R5: 数据持久化

**描述**：背包数据的保存和加载

**技术需求**：
- 与项目的存档系统集成
- Save：将 InventoryData 序列化并保存
- Load：从存档加载 InventoryData
- 清档时也清空背包数据

---

### R6: 物品筛选 & 排序

**描述**：背包内物品的分类和排序

**技术需求**：
- 按类型筛选（Weapon/Armor/Consumable/...）
- 按稀有度排序（从高到低）
- 按名称搜索
