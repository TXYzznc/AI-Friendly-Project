# 01.配置表系统

## 基本信息
- **功能描述**：为框架补充配置表（DataTable）的读取与加载能力。AI 直接编辑 JSON 格式的配置表源文件，通过 Editor 生成器产出强类型 C# 访问类，运行时由 `DataTableModule` 统一加载到内存供业务模块查询。
- **核心目标**：
  1. 解决"AI 编辑配置表困难"的问题——取代旧项目 `xlsx → TSV → xlsx → .cs/.bytes` 的多步中转流程，AI 直接编辑 JSON 即可
  2. 为框架补齐 Category 0 通用配置加载模块（其他模块的初始化可能依赖配置数据）
- **优先级**：高（基础设施，后续业务模块普遍依赖）

---

## 功能清单

### 1. 配置表源文件格式
1.1 JSON 格式，**一表一文件**，文件内同时包含表名(`table`)、字段定义(`fields`：name/type/desc)、数据行(`rows`)
1.2 存放于 `Assets/Resources/DataTable/{TableName}.json`，**源文件 = 运行时资源**，单一位置，无需额外同步步骤
1.3 第一个字段必须是 `Id`（int，主键），用于 `GetById` 查询
1.4 支持类型：`int` `float` `double` `string` `bool` 及对应数组类型（`int[]` 等）
1.5 `rows` 中缺失的字段，生成器按类型默认值处理（`0` / `""` / `false` / `[]`）

### 2. DataTableGenerator（Editor 生成器）
2.1 扫描 `Assets/Resources/DataTable/*.json`
2.2 按 `fields` 生成强类型 Row 类 + Table 容器类（含 `GetById`/`TryGetById`/`All`）
2.3 汇总生成 `DataTableRegistry.cs`（记录所有表的文件名与类型，供运行时加载）
2.4 输出到 `Assets/Scripts/DataTable/*.cs`（生成产物，**不可手改**）
2.5 字段类型不支持、字段名非法等问题 → 报错并中止，不静默跳过
2.6 幂等：重复运行结果一致

### 3. DataTableModule（运行时框架模块）
3.1 `ModuleCategory = 0`（启动基础层）
3.2 `InitializeAsync` 时**一次性**加载并解析全部表（小体量回合制 RPG 配置表，全量加载最简单）
3.3 提供 `GetTable<T>()` 强类型查询接口
3.4 加载失败（文件缺失/JSON 解析错误）→ 抛异常终止启动，遵循 `ModuleRunner` 现有失败语义

### 4. 依赖引入
4.1 添加官方包 `com.unity.nuget.newtonsoft-json`（用户已手动安装）

### 5. 示例表验证
5.1 创建示例表 `ItemTable`（Id/Name/IconId/Tags 等字段），跑通全流程：编辑 JSON → 运行生成器 → 生成 .cs → `DataTableModule` 加载 → `GetTable<ItemTable>().GetById(...)` 查询

## 约束 & 设计决策

| 决策 | 内容 | 原因 |
|---|---|---|
| 数据格式 | JSON（取代 xlsx/TSV/.bytes） | AI 可直接读写编辑，无中转步骤；键值对结构不存在"列对齐"问题 |
| 加载策略 | 启动时一次性全量加载所有表 | 配置表体量小，避免按需加载的时序复杂度 |
| 读取方式 | 生成器产出强类型 Row/Table 类 | 编译期类型检查，符合"不要硬编码数值，配置走 DataTable"的要求 |

### 目录划分
```
Assets/Resources/DataTable/         ← 配置表资源文件（源文件 = 运行时数据）
Assets/Scripts/DataTable/           ← 生成的 Row/Table 类 + 业务扩展解析脚本
Assets/Scripts/Modules/DataTable/   ← DataTableModule + Editor 生成器（框架机制本身）
```

### 不在本次范围内
- xlsx/Excel 相关的任何流程
- 按需加载 / 异步分表加载
- Launch 场景启动流程（另行讨论，不在本功能范围）
